# 01. 기획 (제품 명세)

## 1. 목적

소규모 사업자/사내가 **고객 예약과 문의를 관리**하는 어드민 + 고객 셀프 예약 서비스.
비개발자도 바이브코딩으로 재사용할 수 있는 템플릿이 목표.

## 2. 사용자 유형

| 유형 | 설명 | 권한 |
|------|------|------|
| 비로그인 방문자 | 공개 스케줄만 열람 | 승인된 예약 시간만(개인정보 X) |
| 고객 (customer) | 회원가입한 일반 사용자 | 본인 예약/문의 생성·조회·취소 |
| 관리자 (admin) | 운영자 | 전체 고객·예약·문의·메모 관리 |

> 관리자 승격은 앱 화면에 없고 **DB에서만** 가능 (`06-운영-배포.md` 참고).

## 3. 핵심 기능

### 고객
- **회원가입/로그인**: 이메일 + 비밀번호. 가입 시 이메일 인증 1회.
- **프로필 온보딩**: 최초 로그인 후 이름(필수)·휴대폰(선택) 입력 + 개인정보 안내 동의.
- **예약**: 네이버 예약식 캘린더에서 날짜 → 연속 시간 슬롯(30분 단위, 여러 개 선택 가능) 선택.
- **내 예약**: 상태 확인, **예약 취소**(요청/승인 상태일 때).
- **문의**: 등록, 목록, 상세에서 관리자 답변 확인.

### 관리자
- **대시보드**: 고객 수, 대기 예약, 미답변 문의 등 요약.
- **고객 관리**: 목록, 상세(예약·문의 이력 + 내부 메모 작성).
- **예약 관리**: 상태 변경(승인/거절/취소/완료/노쇼).
- **문의 관리**: 답변 작성 + 상태 변경.
- **내부 메모**: 고객별 비공개 메모 (고객에게 절대 노출 안 됨).

### 공개
- **공개 스케줄**: 월간 캘린더. 승인된 예약이 있는 날 표시 + 마우스 오버 시 시간 툴팁.
  개인정보(이름/이메일/전화/요청사항)는 **절대 노출 안 함**.

## 4. 화면 목록

### 고객/공개
- `/login` — 로그인/회원가입 탭
- `/onboarding` — 최초 프로필 설정
- `/my` — 내 정보 홈
- `/my/edit` — 프로필 수정(이름·휴대폰만)
- `/my/appointments` — 내 예약 목록 (+ 취소)
- `/my/appointments/new` — 예약하기(주간 현황표 + 캘린더 + 슬롯 다중선택)
- `/my/contact`, `/my/contact/new`, `/my/contact/[id]` — 문의 목록/등록/상세
- `/schedule` — 공개 월간 스케줄
- `/privacy` — 개인정보 처리방침

### 관리자
- `/admin` — 대시보드
- `/admin/customers`, `/admin/customers/[id]` — 고객 목록/상세
- `/admin/appointments` — 예약 관리
- `/admin/contact`, `/admin/contact/[id]` — 문의 관리/답변

## 5. 핵심 사용자 플로우

### 회원가입 → 첫 예약
1. `/login` 회원가입 탭에서 이메일+비밀번호 → 인증 메일 발송
2. 메일의 인증 링크 클릭 → `/api/auth/callback` → 세션 생성 → `/onboarding`
3. 이름 입력 + 개인정보 동의 → 프로필 생성(role=customer) → `/my`
4. 예약하기 → 날짜 선택 → 연속 슬롯 선택 → 예약 신청(status=requested)
5. 관리자가 승인 → 고객 예약 목록에 "승인" 표시, 공개 스케줄에 노출

### 재방문 로그인
- `/login` 로그인 탭 → 이메일+비밀번호 → 끝 (이메일 발송 없음)

## 6. 예약 슬롯 규칙 (기본값, 조정 가능)

`src/lib/booking.ts`:
- 슬롯 길이 `SLOT_MINUTES = 30`
- 영업시간 `BUSINESS_START_HOUR = 9` ~ `BUSINESS_END_HOUR = 18`
- 예약 가능 기간 `BOOKING_RANGE_DAYS = 60` (오늘부터 60일)
- 1회 최대 연속 `MAX_BOOKING_SLOTS = 8` (8 × 30분 = 4시간)

## 7. 의도적으로 제외한 것 (MVP 범위)

- 결제, 알림(문자/푸시), 마케팅 동의
- 영업일/휴무일/시간대별 예약 정원 (전부 단일 정원 = 슬롯당 1건)
- 관리자 승격 UI (보안상 DB에서만)
