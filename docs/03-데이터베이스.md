# 03. 데이터베이스

마이그레이션: `supabase/migrations/20260628000000_init_vibe_admin_schema.sql`
대시보드 붙여넣기용(재실행 안전): `supabase/setup-database.sql`

## 1. ENUM

- `user_role`: `customer` | `admin`
- `appointment_status`: `requested` | `approved` | `rejected` | `cancelled` | `completed` | `no_show`
- `contact_status`: `open` | `in_progress` | `answered` | `closed`

## 2. 테이블

### profiles — 고객/관리자
| 컬럼 | 타입 | 비고 |
|------|------|------|
| id | uuid PK | = auth.users.id (FK 없음) |
| role | user_role | 기본 customer, 고객은 변경 불가 |
| email | text unique | 서버에서 auth user 기준 저장 |
| name | text | 고객 입력 |
| phone | text | 선택. 미입력 시 null |
| privacy_agreed_at | timestamptz | 서버 시각 |
| created_at / updated_at | timestamptz | |

### appointments — 예약
| 컬럼 | 타입 | 비고 |
|------|------|------|
| id | uuid PK | |
| customer_id | uuid | FK 없음, = auth.uid() (서버 강제) |
| customer_email / customer_phone | text | 생성 시점 스냅샷 |
| customer_message | text | 요청사항 |
| scheduled_at | timestamptz | 슬롯 시작 |
| duration | integer | 분. 연속 슬롯 수 × 30 |
| status | appointment_status | 기본 requested |
| admin_message | text | 관리자 안내 |
| approved_at / cancelled_at / completed_at | timestamptz | 상태 전환 시각 |

### contact — 문의
`id, customer_id, customer_email, customer_phone, status(open), title, body, admin_reply, answered_at, created_at, updated_at`
(컬럼명 `answered_at` — 오타 `answred_at` 주의)

### customer_notes — 관리자 내부 메모
`id, customer_id, customer_email, customer_phone, admin_id, title, body, created_at, updated_at`
**고객에게 절대 노출 안 됨.**

## 3. 인덱스 (조회 화면 기준)

```
profiles(role), profiles(email)
appointments(customer_id), (status, scheduled_at), (created_at desc)
customer_notes(customer_id), (admin_id)
contact(customer_id), (status, created_at desc)
```

## 4. 트리거

`set_updated_at()` → 4개 테이블 update 시 `updated_at` 자동 갱신.

## 5. RLS

모든 테이블 `enable row level security`. anon select 정책 없음(공개 데이터는 서버 API로만).

### is_admin() 헬퍼
```sql
create function public.is_admin() returns boolean
language sql security definer stable set search_path = public
as $$ select exists(select 1 from public.profiles where id = auth.uid() and role='admin') $$;
```
`security definer` + `search_path` 고정으로 RLS 재귀 방지. authenticated에 execute 부여.

### 정책 요약
| 테이블 | 고객 | 관리자 |
|--------|------|--------|
| profiles | 본인 select / insert(role=customer 강제) / name·phone update | 전체 select·update (is_admin) |
| appointments | 본인 select / insert(status=requested 강제) | 전체 select·update |
| contact | 본인 select / insert(status=open 강제) | 전체 select·update |
| customer_notes | **불가** | 전체 select·insert·update·delete |

### RLS만으로 부족한 부분 → 서버 API로 보강
- 컬럼 단위 제한(role/email/status/admin_reply 변경 금지)은 RLS update 정책만으로 완전치 않음 →
  고객용 변경은 **서버 API에서 허용 필드만** 처리하거나 경로 자체를 안 만듦.
- 예약 취소는 고객 update 정책을 열지 않고, `/api/appointments/[id]/cancel`에서
  소유권·상태 검증 후 service role로 제한 수행.
