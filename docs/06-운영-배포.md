# 06. 운영 · 배포

## 1. 환경변수 (`.env.local` / Vercel)

| 변수 | 설명 | 위치 |
|------|------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase 프로젝트 URL | 클라/서버 |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | anon(publishable) 키 — 공개 OK | 클라/서버 |
| `SUPABASE_SERVICE_ROLE_KEY` | service role(secret) 키 — **서버 전용, 비밀** | 서버만 |
| `NEXT_PUBLIC_APP_URL` | 앱 URL (배포 도메인) | 클라/서버 |
| `DEFAULT_TIMEZONE` | 기본 타임존(Asia/Seoul) | 서버 |

- 새 키 형식: anon=`sb_publishable_...`, service role=`sb_secret_...`.
- **service role 키에 `NEXT_PUBLIC_` 절대 금지.**
- Vercel 등록은 값 노출 없이: `grep '^KEY=' .env.local | cut -d= -f2- | vercel env add KEY production`

## 2. 첫 배포 순서

```bash
# 1) DB 스키마
supabase link --project-ref <ref>
supabase db push     # 또는 대시보드 SQL Editor에 supabase/setup-database.sql 붙여넣기

# 2) Auth URL 설정 (CLI)
#    config.toml의 site_url / additional_redirect_urls 를 운영 도메인으로 두고:
supabase config push --project-ref <ref>

# 3) Vercel
vercel link --yes
# 환경변수 4종 등록 후
vercel --prod --yes
```

- `vercel.json`에 `{"framework":"nextjs"}` 명시(정적 사이트 오인 방지).
- Next.js는 보안 패치 버전 유지(취약 버전은 Vercel이 배포 차단).

## 3. Supabase Auth URL 설정

`supabase/config.toml`:
```
site_url = "https://<운영도메인>"
additional_redirect_urls = ["https://<운영도메인>/**", "http://localhost:3000/**"]
```
→ `supabase config push`. 안 하면 인증 링크가 엉뚱한 곳으로 가거나 `error=auth` 발생.

## 4. 첫 관리자 만들기

앱에 승격 UI 없음. 대상 계정이 **로그인+프로필 생성**을 끝낸 뒤:

```sql
-- Supabase SQL Editor
update public.profiles set role='admin' where email='운영자이메일';
```
또는 service role 스크립트로 동일 처리.

## 5. 이메일 발송 (SMTP)

- 무료 플랜 **기본 이메일**: 발송량 매우 적음(테스트용). 운영 부적합.
- **로그인은 이메일을 안 보냄**(비밀번호) → 이메일은 **회원가입 인증 시 1회**만.
- 운영 시 **커스텀 SMTP**(Resend/SendGrid/SES 등) 연결 권장:
  - 연결하면 발송량 증가 + 한글 인증 메일 템플릿 사용 가능.
  - SMTP 연결 후 `config.toml`의 `[auth.email.template.confirmation]` 주석을 풀고
    `supabase/templates/confirmation.html`(token_hash 방식)로 push → 브라우저 무관 인증.

## 6. 운영자가 수정해야 하는 값

| 항목 | 위치 |
|------|------|
| Supabase 키 / 앱 URL | `.env.local`, Vercel 환경변수 |
| 개인정보 처리방침 사업자 정보 | `src/app/privacy/page.tsx` → `OPERATOR_INFO` |
| 개인정보 기준일 | `src/app/onboarding/page.tsx`, `privacy/page.tsx` |
| 영업시간/슬롯/예약범위 | `src/lib/booking.ts` 상단 상수 |
| Auth Site URL/Redirect | `supabase/config.toml` → `config push` |

## 7. 로컬 개발

```bash
bash scripts/preflight.sh    # 도구/로그인/env 점검
cp .env.example .env.local   # 값 채우기
npm install
npm run dev
```
Windows는 Git Bash/WSL 권장.
